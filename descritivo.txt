🐶 TOBBY - DOCUMENTAÇÃO COMPLETA DO PROJETO
Versão 5.0 - Com Criptografia AES-256
📌 1. VISÃO GERAL DO PROJETO
Tobby é um organizador financeiro pessoal com personalidade, projetado para ser um "companheiro financeiro". O nome é uma homenagem ao cachorro Tobby Maguire.

Filosofia
Não é um app bancário (não faz transferências, pagamentos reais)

É um organizador e acompanhante financeiro

Foco em personalidade e conexão emocional com o usuário

IA que conversa de forma natural e personalizada

Privacidade em primeiro lugar (dados criptografados)

🗄️ 4. BANCO DE DADOS (SUPABASE)
Script SQL Completo para Criar Todas as Tabelas
sql
-- ============================================
-- BANCO DE DADOS DO TOBBY - CRIAÇÃO COMPLETA
-- Versão 5.0 com Criptografia
-- ============================================

-- ============================================
-- 1. TABELA DE USUÁRIOS (users)
-- Nota: Esta tabela já existe no Supabase Auth
-- Apenas adicionamos colunas extras
-- ============================================

-- Adicionar colunas extras na tabela users (se não existirem)
ALTER TABLE users 
ADD COLUMN IF NOT EXISTS salary_encrypted TEXT,
ADD COLUMN IF NOT EXISTS reset_token TEXT,
ADD COLUMN IF NOT EXISTS reset_expires TIMESTAMP;

-- ============================================
-- 2. TABELA DE CONTAS (bills)
-- ============================================
DROP TABLE IF EXISTS bills CASCADE;
CREATE TABLE bills (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  name VARCHAR(200) NOT NULL,
  value_encrypted TEXT NOT NULL,
  due_day INTEGER NOT NULL CHECK (due_day BETWEEN 1 AND 31),
  category VARCHAR(50) DEFAULT 'outros',
  status VARCHAR(20) DEFAULT 'pending' CHECK (status IN ('pending', 'paid', 'late')),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- 3. TABELA DE INVESTIMENTOS (investments)
-- ============================================
DROP TABLE IF EXISTS investments CASCADE;
CREATE TABLE investments (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  symbol VARCHAR(20) NOT NULL,
  asset_type VARCHAR(20) DEFAULT 'stock' CHECK (asset_type IN ('stock', 'fii', 'crypto', 'fixed_income')),
  quantity DECIMAL(10,4) NOT NULL,
  purchase_price_encrypted TEXT NOT NULL,
  purchase_date DATE NOT NULL,
  current_price_encrypted TEXT,
  broker VARCHAR(50),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- 4. TABELA DE FINANCIAMENTOS (loans)
-- ============================================
DROP TABLE IF EXISTS loans CASCADE;
CREATE TABLE loans (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  name VARCHAR(100) NOT NULL,
  type VARCHAR(20) DEFAULT 'personal' CHECK (type IN ('mortgage', 'car', 'personal', 'student', 'credit_card')),
  total_principal_encrypted TEXT NOT NULL,
  outstanding_balance_encrypted TEXT NOT NULL,
  interest_rate DECIMAL(8,4) NOT NULL,
  monthly_payment_encrypted TEXT,
  remaining_installments INTEGER,
  amortization_type VARCHAR(10) DEFAULT 'price' CHECK (amortization_type IN ('sac', 'price')),
  start_date DATE NOT NULL,
  extra_payment DECIMAL(10,2) DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- 5. TABELA DE BENS/PATRIMÔNIO (assets)
-- ============================================
DROP TABLE IF EXISTS assets CASCADE;
CREATE TABLE assets (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  name VARCHAR(100) NOT NULL,
  asset_type VARCHAR(30) DEFAULT 'other' CHECK (asset_type IN ('real_estate', 'vehicle', 'savings', 'other')),
  estimated_value_encrypted TEXT NOT NULL,
  acquisition_value_encrypted TEXT,
  acquisition_date DATE,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- 6. TABELA DE TICKETS (meal_vouchers) - Benefícios
-- ============================================
DROP TABLE IF EXISTS meal_vouchers CASCADE;
CREATE TABLE meal_vouchers (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  voucher_type VARCHAR(20) CHECK (voucher_type IN ('meal', 'food')),
  provider VARCHAR(50),
  monthly_amount DECIMAL(10,2) NOT NULL,
  current_balance DECIMAL(10,2) NOT NULL,
  reset_day INTEGER DEFAULT 1,
  last_reset DATE,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- 7. TABELA DE MEMÓRIA DA IA (conversation_memory)
-- ============================================
DROP TABLE IF EXISTS conversation_memory CASCADE;
CREATE TABLE conversation_memory (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  key VARCHAR(100),
  value TEXT,
  confidence DECIMAL(3,2),
  last_updated TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- 8. TABELA DE TRANSAÇÕES DE TICKETS (voucher_transactions)
-- ============================================
DROP TABLE IF EXISTS voucher_transactions CASCADE;
CREATE TABLE voucher_transactions (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  voucher_id UUID NOT NULL REFERENCES meal_vouchers(id) ON DELETE CASCADE,
  amount DECIMAL(10,2) NOT NULL,
  establishment VARCHAR(100),
  transaction_date DATE NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

-- ============================================

-- ============================================
-- VERIFICAR TABELAS CRIADAS
-- ============================================
SELECT 
  table_name, 
  (SELECT COUNT(*) FROM information_schema.columns WHERE table_name = t.table_name) as column_count
FROM information_schema.tables t
WHERE table_schema = 'public' 
  AND table_name IN ('users', 'bills', 'investments', 'loans', 'assets', 'meal_vouchers', 'conversation_memory')
ORDER BY table_name;
🔐 5. CRIPTOGRAFIA
Como Funciona
Algoritmo: AES-256-CBC

Chave: 32 bytes (64 caracteres hex) via variável ENCRYPTION_KEY

Campos criptografados: Todos os valores financeiros sensíveis

Campos Criptografados por Tabela
Tabela	Campos Criptografados
users	salary_encrypted
bills	value_encrypted
investments	purchase_price_encrypted, current_price_encrypted
loans	total_principal_encrypted, outstanding_balance_encrypted, monthly_payment_encrypted
assets	estimated_value_encrypted, acquisition_value_encrypted
Gerar Chave de Criptografia
bash
# Node.js
node -e "console.log(crypto.randomBytes(32).toString('hex'))"

# Ou use: https://www.random.org/cgi-bin/randbyte?format=hex&nbytes=32

🐛 11. PROBLEMAS CONHECIDOS E SOLUÇÕES
Problema	Causa	Solução
Supabase desconectado	URL com /rest/v1/ no final	Remover o caminho extra da URL
Erro PGRST125	Tabela não encontrada	Verificar nome da tabela (users, não tobby_users)
Timeout no Supabase	Projeto gratuito hiberna	Primeira requisição após pausa demora alguns segundos
Criptografia não funciona	ENCRYPTION_KEY não configurada	Adicionar variável no Render
Erro 401 no admin	Chave ADMIN_KEY incorreta	Usar tobbyadmin2024 no header x-admin-key

🎯 14. RESUMO PARA IA
Tobby é um organizador financeiro pessoal com personalidade (mascote cachorro). Backend Node.js com Express e Supabase (PostgreSQL). Frontend PWA puro (HTML/CSS/JS). Autenticação JWT. Criptografia AES-256 para dados sensíveis. Funcionalidades: controle de contas, investimentos (ações com cotações em tempo real via Brapi e CoinGecko), financiamentos (simulação de amortização), patrimônio (bens), leitura de nota fiscal por foto, leitura de holerite, importação de extrato (CSV/OFX/texto), informe de IR, chat com IA (Claude API). Design responsivo com tema claro/escuro. Hospedado no Render (backend) e GitHub Pages (frontend). Tabela principal: users (do Supabase Auth) com colunas extras para criptografia. Tabelas de dados: bills, investments, loans, assets, meal_vouchers, conversation_memory. Variáveis obrigatórias: SUPABASE_URL, SUPABASE_SECRET_KEY, JWT_SECRET, ADMIN_KEY. Opcional: ENCRYPTION_KEY (para criptografia), CLAUDE_API_KEY (para IA).

Com este documento, qualquer IA pode entender exatamente o que foi construído e continuar o desenvolvimento! 🐶📚
